# Security Policy

## Supported versions

ClearRead is currently pre-1.0 style and maintained on the latest `main` branch.
Please report vulnerabilities against the latest commit.

## Reporting a vulnerability

If you discover a security issue, please do not open a public issue first.

Instead, report it privately by contacting the maintainers with:

- A clear description of the issue
- Steps to reproduce
- Affected components (`frontend`, `backend`, or both)
- Potential impact
- Suggested remediation (if available)

The maintainers will acknowledge reports as quickly as possible and work on a fix.

## Disclosure process

1. Report is received and triaged.
2. Maintainers validate and prepare a fix.
3. Patch is released.
4. Public disclosure follows after users have reasonable time to update.
